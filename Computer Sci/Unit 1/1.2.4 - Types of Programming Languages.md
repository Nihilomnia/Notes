# Modes of Addressing Memory 

1.  Immediate addressing - sometimes called an immediate operand, the value in the address  is actually the value to be used, so the memory does not need to be searched to find the required value. for example ADD 10 means add 10 not add the value in address 10
   
2. Direct addressing - The value in the address part of the instruction is a reference to the address in memory where the required value is located. for example ADD 10 would mean get the value of memory wardress 10 and add it to the value in the accumulator.
   
3. Indirect addressing - The value in the address part if the instruction is a reference to a memory location that contains the address in memory where the required value is located. for example ADD 10 would mean got o memory address 10  then you have go to the memory address that was there and add the value of the second address.
   This mode of addressing  is useful as it means larger address ranges can be used to reference data and instructions.
   
   For example with 32 bit instructions which 16 bits for the opt code and 16 bots for the operand 
   
   we can gave a total of 65536 total combinations for instructions (2^16) however this means that would would be hard stuck at that amount of memory  by using Indirect dressing allows to to get to addresses way beyond the 65536  
   
4.  Indexed Addressing uses and index  register  (IR). The IR is set to 0, so the first value taken is the index + the value given for example ADD 10 with and Index of 0 would mean go to the memory address location 10 and load the value in address 10


   
# Object-Oriented Programming (OOP)

Object-Orientated-programming (OOP) attempts to capture or group information, data and related functionality (code) into structured items known as Objects.

With OOP, the world is viewed as a collection of objects.

An object could be anything such as an animal,car, person or house, it also possible for it to be abstract constructs such as a customer's account or a data structure. 

Objects interact with each other by sending and receiving messages.

Encapsulation - "The building of data with the methods that operate on and restrict direct access to it"


Encapsulation is used to hide the values or internal state of an object preventing direct access bu unauthorized  parties.

Encapsulated attributes of an objects should only be accessible or changeable via the methods provided by the objects.

Encapsulation helps keep data related to an object safe - it can't accidentally be altered by another part of the program  with out using the code proved by the methods, keeping the programmer in control.
